F.A.Q

Frequently Asked Questions

Below are some of the most frequently asked questions regarding placements and interviews:

  • What programming languages should I learn?

    You can learn the many programming languages ​​you want but make sure you are familiar with any single programming language. Employers only see if you can code a particular problem in any language you like (C, C ++, Java etc.). We recommend that you learn any formal programming languages ​​such as C and any Object Oriented Programming languages ​​such as C ++, Java etc.

  • Is competitive programming enough for placements?

    Competitive Programming plays a veritably important part in developing problem- working chops and the capability to perform under the pressure of time. But honestly, it isn't enough to prepare you from the point of view of placement. This is because companies concentrate on content data structure and algorithm generalities and other important CS assignments. You need to have a good understanding of generalities with good problem- working chops (as demanded to break some of the real life problems during your career), good practice of rendering.

  • How should I use TechMentor for placement preparation?

    Exercise core CS Fundamentals like DBMS, OOPs, OS, Networking and Aptitude Questions. Data Structures and Algorithms like Stack, Queue, Linked List, Tree, Heap, Graph, DP are many DS which is veritably common and you should know ins and outs of them irrespective of company you're targeting. Start these motifs in the order we've mentioned. Break many questions of each and also you'll start observing patterns. For illustration, Utmost of the tree questions boils down to working it in three corridor, Solve left subtree, Solve right subtree, combine the results.

  • What exactly big tech giants are looking for in the candidates?

    Tech recruiters often take a deeper look at the projects you've worked on to see how you built solutions with the available tools. They judge you according to your problem solving and team work skills They're going to screen your portfolio, competitive profiles and open source contributions to determine your role in various projects as an individual contributor or as part of a team

  • What to do during Onsite technical interview and whiteboard challenge?

    1. Ask clarifying questions before you even start writing code.
    2. Talk through your code (think out loud) to give the interviewer a window into your thoughts.
    3. Speak clearly and precisely.
    4. If you’re stuck, stay calm, think out loud, and try new ideas.
